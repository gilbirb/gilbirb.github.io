1. What is a *branch delay?*

The R3000 uses a 5-stage pipeline: IF → RD → ALU → MEM → WB. When the CPU fetches a branch instruction, it has already begun fetching the next instruction before it knows where to jump. The instruction immediately following a branch always executes, regardless of whether the branch is taken.

5. Compare cooperative versus preemptive multithreading?

Cooperative Multithreading
* Threads voluntarily yield control. Thread has to explicitly give up the CPU.
* (+) Context switches are predictable, as defined by the programmer.
* (-) One thread can starve others if it never yields.

Preemptive Multithreading
* The scheduler forcibly interrupts the threads.
* (+) No single thread can monopolise the CPU

6. Describe *user-level threads* and *kernel-level threads*. What are the advantages or disadvantages of each approach?

User level threads:
* Pros:
  * Thread management and switching at user level is much faster than doing it in kernel level
  * Can customize scheduling for your specific program, instead of being stuck with the OS's generic scheduling rules.
  * Can be implemented on any OS (thread or non-thread aware)
  * Can easily support massive numbers of threads on a per-application basis
* Cons:
  *  Threads have to yield() manually
    * A single poorly implemented thread can monopolise the available CPU time
  * Does not take advantage of multiple CPUs
  * If a thread makes a blocking system call, the process (and all the internal threads) blocks

Kernel level threads:
* Pros:
  * Preemptive multithreading
  * Parallelism
    *  Can overlap blocking I/O with computation
    * Can take advantage of a multiprocessor
* Cons:
  * Thread creation and destruction, and blocking and unblocking threads requires kernel entry and exit. (More expensive than user-level equivalent)

7. A web server is constructed such that it is multithreaded. If the only way to read from a file is a normal blocking read system call, do you think user-level threads or kernel-level threads are being used for the web server? Why?

Kernel-level threads must be used because a blocking system call causes the OS to block the entire process when using user-level threads, freezing all threads, whereas with kernel-level threads the OS can keep other threads running while one waits on I/O.
